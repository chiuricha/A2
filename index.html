Welcome to the Helicopter Game!

This helicopter game was developed using Javascript. I explored 2 different approaches: Phaser.js, and pure Javascript with HTML5. I was able to understand the benefits and trade offs of using a game development framework vs coding everything from scratch.

I chose to make this helicopter game because it was one of my favourite childhood games in the 2000s. This type of game is addicting due to its physical based movement and obstacle avoidance. However the physics, gravity, acceleration, collision detection, and score tracking are what should be taken into considerations when designing and coding. Building this game by following an existing tutorial was fun and valuable.

I finalized the game using pure JavaScript and the HTML5 Canvas, without using an external framework (Phaser.js), because using Phaser.js could not achieve the same kind of physics as the original helicopter game. I actually find it more fun creating from scratch and following tutorials, due to the fact that it has better flexibility where it gives me full control over the game mechanics. Unlike Phaser.js, where movement and physics are built in, and it simplified the implementation of gravity and acceleration. This is the main reason I decided not to use Phaser.js, because it just could not achieve the type of acceleration of a helicopter I wanted(When user pressing spacebar). I had to manually code physics, collision detection, and user input for controlling helicopters. Though it was time consuming, by following the tutorial, I was able to precisely adjust the movement of the helicopter, manage the obstacles, and create the tunnel(walls). And in Phaser.js, these would be automated with limited customization.

In the future, I could integrate sound effects, more animations (when collision happens), moving obstacles, or gradually increase the difficulty option. And to make things easier, I could use Howler.js for sound effects, and Lodash for utility functions.
